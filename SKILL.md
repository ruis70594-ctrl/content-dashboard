# 内容看板系统

帮用户管理多平台内容。**底层 markdown = 给 AI 读的语料，上层 HTML = 给人看的可视化看板。一份内容两个面孔，HTML 由 markdown 数据生成。**

## 系统位置与结构

默认在用户项目下的 `内容库/` 文件夹（若不在，先问或找 `内容看板.html` 所在目录）：

```
内容库/
  内容看板.html      # 给人看的网站，单文件离线可双击
  _模板.md           # 新稿子复制母版
  小红书/ 抖音/ 公众号/ 淘宝咸鱼/   # 每篇稿子一个 .md
  对标库/            # 别人的、要学要抄的帖子
  封面/              # 封面原图（用户手动丢进来）
```

首次搭建：把本 skill 目录的 `template.html` 复制成 `内容库/内容看板.html`。

## 分工铁律

**用户只负责：写稿、发截图、报数据、把封面图丢进 `封面/` 文件夹。代码和归档全由你做，用户绝不碰 HTML 代码**（手改易出错，且用户怕"维护成本高就弃坑"）。

## 加一篇稿子（用户发来已发笔记的截图）

1. 从截图读出：标题、正文、浏览/点赞/收藏/评论、TAG、评论区内容、发布日期、是自然流还是投流。
2. 在对应平台文件夹建 `.md` 文件，命名 `日期_产品_类型_主题.md`（文件名即定位用的"身份证"），frontmatter 见 `_模板.md`。
3. 在 `内容看板.html` 的 `POSTS` 数组加一个对象（字段见 README 的数据结构）。
4. 封面：用户把图丢进 `封面/` 后，用 PowerShell 转 base64 内嵌进 HTML（保证整个文件可传手机/分享不裂图）：
   ```powershell
   $b=[Convert]::ToBase64String([IO.File]::ReadAllBytes("封面/xxx.webp"))
   # 把 POSTS 里该篇的 封面 字段写成 "data:image/webp;base64,$b"
   ```
   原图留在 `封面/` 当备份；HTML 里放内嵌版。
5. md 和 HTML 两处都要同步。

## 评级规则（自动判，不让用户判断）

默认按小红书自然流浏览量（**可按用户账号体量调整，问一次基准就定死**）：

- 浏览 < 200 → 扑街（没推起来）
- 浏览 200–2000 → 普通
- 浏览 2000+ → 爆款
- 例外：赞藏破 20 或评论破 10，即使浏览没到 2000 也抬成爆款（互动比浏览值钱）
- **投流的单独标"投流"标签，不参与浏览判断**（浏览是买来的），改看花钱有没有换来互动/进私
- 浏览异常低（如个位数）多半是**限流**而非文案差，要在备注里点明，别污染"什么文案好用"的判断

## 更新数据 / 单篇折线

用户报新数据时，定位：靠 .md 文件名或标题；多篇同名时先报标题让用户确认，别瞎改。更新动作：
1. 往该篇 `数据历史` / `历史` 数组加一个带日期的新快照
2. 把 `数据` 改成最新值
3. 按规则重判评级
4. md 和 HTML 同步

弹窗内有单篇"浏览走势"SVG 折线（≥2 点连线，1 点显示提示）。

## "更新看板"

= 扫描所有 .md 文件，重新生成 HTML 里的 `POSTS`/`REFS` 数据和图表。生成后提示用户双击打开/刷新。

## 对标库

别人的帖子存 `对标库/` + HTML 的 `REFS` 数组，和用户自己的内容分开（两个标签页）。重点记"想学的点（亮点）"。

## 语料闭环（这套系统的终极价值）

评级=爆款的稿子是喂内容生成 skill 的**正面语料**，扑街的是**反面教材**。用户说"学一下我的爆款"时，读爆款稿提炼规律，沉淀回对应的内容 skill（如小红书/咪蒙类 skill 的风格学习日志）。
