# 内容看板 · Content Dashboard

把你在多平台（小红书 / 抖音 / 公众号 / 淘宝咸鱼）发布的内容，归档成一个**既能一目了然管理、又能反复喂回 AI 当语料**的本地系统。

> 一句话：**底层是 markdown 文件（给 AI 读），上层是一个单文件离线 HTML 看板（给人看）。一份内容两个面孔。**

这是一个 [Claude Code](https://claude.com/claude-code) skill。装上之后，你只管写稿、发截图、报数据，归档和看板维护全交给 AI——你一行代码都不用碰。

## 它能干嘛

- 📥 **甩张截图就归档**：把已发笔记截图发给 AI，自动读出标题、正文、数据、标签、评论，建档入库
- 📊 **可视化看板**：每周互动趋势、平台分布、评级分布；每篇一张卡片，带封面、数据、TAG
- 📈 **单篇浏览走势折线**：看一篇内容怎么慢慢起量
- 🏆 **自动评级**：爆款 / 普通 / 扑街，按浏览量客观判定，投流单独算
- 🎯 **对标库**：把别人的、想学想抄的帖子单独收一处
- 📦 **单文件可分享**：封面图焊进 HTML，整个看板一个文件，能双击、能传手机、能发给合伙人，不裂图
- ♻️ **语料闭环**：爆款当正面语料、扑街当反面教材，喂回 AI 持续学你的风格

## 安装

把 `content-dashboard/` 文件夹放进 `~/.claude/skills/`（或项目的 `.claude/skills/`）。重启 Claude Code 即可。

## 用法

```
你：把这篇加进内容库  （附一张已发笔记的截图）
你：更新看板
你：那篇 grok 的浏览涨到 1200 了
你：把这个想抄的帖子存进对标库  （附截图）
你：学一下我最近的爆款
```

## 数据结构

每篇稿子在 HTML 的 `POSTS` 数组里是一个对象：

```js
{
  平台:"小红书", 类型:"软广", 产品:"Gemini家庭组", 日期:"2026-06-09",
  状态:"已发", 评级:"爆款", 来源:"自然流",
  标题:"...", 封面:"data:image/webp;base64,...",
  标签:["...","..."],
  数据:{浏览:32000,点赞:1800,收藏:960,评论:140},
  历史:[{日期:"2026-06-09",浏览:3,点赞:1,收藏:1,评论:0}, ...],  // 折线数据
  备注:"为什么爆/为什么扑，一句话",
  评论:["评论区摘录","..."],
  正文:"..."
}
```

底层每篇还有一个对应的 `.md` 文件（带 frontmatter），那是喂 AI 的语料。

## 评级规则（可按账号体量调整）

| 自然流浏览量 | 评级 |
|---|---|
| < 200 | 扑街 |
| 200–2000 | 普通 |
| 2000+ | 爆款 |

赞藏破 20 或评论破 10 → 抬成爆款。投流单独标记，不参与浏览判断。
